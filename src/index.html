What is vibe coding?

          

            Vibe coding is building software by describing what you want in plain language and letting AI write the actual code.
            You focus on the idea — the features, the feel, the goal — while the AI handles the implementation details.
            You don't need to memorize syntax or language rules. You need to communicate clearly.
          

        


        

          
Setting up your AI

          
    
            
  1. Choose a provider — Claude (Anthropic), ChatGPT (OpenAI), or Gemini (Google). All three work well. Claude tends to be strongest for long coding tasks.
    2. 
            
  2. Create an account at their website: claude.ai, platform.openai.com, or aistudio.google.com.
    3. 
            
  3. Generate an API key — look for "API keys" or "API access" in your account settings.
    4. 
            
  4. Paste the key into the Settings tab in this panel. It's stored encrypted on your device.
    5. 
            
  5. Start chatting in the Chat tab. Your messages go directly to the AI — no middleman.
    6. 
          

        


        

          Context files — tell the AI about your project
          

            

              Context files are plain text files that sit in your project folder.
              When you open that folder in an AI-aware editor, the AI reads them automatically and understands your project without you explaining it every time.
            


            
CLAUDE.md — for Claude

            
# Project
A personal portfolio website.

# Stack
HTML, CSS, vanilla JavaScript. No frameworks.

# Style preferences
- 2-space indentation
- Prefer CSS custom properties over hardcoded values
- Mobile-first responsive design

# Avoid
- jQuery
- Inline styles

          Writing good prompts — 5 tips
          

            
    
              
  1. 
                Specify the language and framework.
                "Write a function" is vague. "Write a TypeScript function using the Fetch API" is specific.
              
    2. 
              
  2. 
                Describe the expected output.
                Say what the result should look like: "Return a JSON array of objects with name and id fields."
              
    3. 
              
  3. 
                Give the AI a role.
                Starting with "You are a senior backend engineer…" shifts the response toward expert-level output.
              
    4. 
              
  4. 
                Include constraints and edge cases.
                "Handle empty input gracefully" or "must work in IE11" prevents surprises.
              
    5. 
              
  5. 
                One task per prompt.
                Split complex work into steps. A focused prompt gets a focused response.

          Understanding the Prompt tab
          

            

              The Prompt tab analyzes your prompt as you type and scores it from 0–100.
              It checks for: missing language context, vague verbs, no expected output specified, prompt length, and role framing.
              Use it before sending important prompts to catch weak spots before the AI sees them.
